## Lab Directions

This lab is open-ended. You will build a complete game using the **PyGame** library. This builds on examples shown in lecture; a minimal example is provided in the `src/` folder to help you get started.

### Assignment

Your assignment is to **create a working game using PyGame**, meeting at least the minimum viable product below.

#### Minimum Viable Product (60%)
1. The game window opens and displays a basic scene.
2. The user can control an on-screen object (e.g., a player sprite) using keyboard input.
3. There is at least one simple game mechanic (e.g., collision, scoring, timer, or movement constraint).

You are strongly encouraged to **be creative and build on this foundation**. Below are additional enhancements you can implement to earn a higher score.

---

## Rubric: Part 1 (this week, in-class)

- **50 pts** – Participation and progress during lab  
- **20 pts** – Game window launches with visible scene  
- **20 pts** – Basic interaction (e.g., object moves with keypress)  
- **10 pts** – At least one game mechanic implemented (collision, timer, boundary, etc.)

> Demonstration is due **prior to next lab session**.

---

## Rubric: Part 2 (next week, in-class)

- **15 pts** – Presence and attentiveness during peer demonstrations  
- **10 pts** – Code style and instructor evaluation  
- **25 pts** – Fully functional game (start, play, end/game-over or win condition)  
- **12 pts** – In-class demonstration of your game  
  - Describe gameplay and features  
  - Show working MVP

> A minimal functional game with good effort yields a **passing grade (62%)**.

## Bonus Features (up to +50 pts)

You may implement **any combination** of the following features to improve your score:

- +10 – Background music or sound effects  
- +10 – Score tracking with on-screen display  
- +10 – Multiple levels or increasing difficulty  
- +10 – Game-over and restart functionality  
- +10 – Sprite animation (e.g., player walks)  
- +10 – Collision detection with multiple objects  
- +10 – Title screen or menu with start button  
- +10 – High score saved to file  
- +10 – Custom graphics or sprite art  
- +10 – Instructor discretion for exceptional work
